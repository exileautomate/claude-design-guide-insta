W1

Understanding the Interface & Your First Design

Get familiar with the tool, understand how it thinks, and build your first real design

How the Tool Works

Claude Design has two areas: a chat interface on the left where you describe what you want, and a live canvas on the right where Claude builds it in real time. The flow is: describe → review on canvas → iterate until it's right → export. Your first few sessions should be about exploring this loop, not producing perfect output.

Task 01 — Day 1

Open Claude Design & Take the Tour

Go to claude.ai/design and create a new project. Don't design anything yet. Spend 15 minutes clicking around. Understand where the chat lives, what the canvas looks like when empty, and where the Export button is. Read the button labels. This orientation will save you confusion later.

💡 Tip
Before you prompt anything, notice the chat panel on the left and the canvas on the right. Every design you build will live on that canvas.
✓ Done When
You can describe where each main feature lives without looking.
Task 02 — Day 1

Build Your First Design in Under 5 Minutes

Use the exact prompt below for your very first design. Don't overthink it. Type it, hit send, and watch Claude work on the canvas. The goal here is just to see the tool in action for the first time.

W2

Mastering Iteration — Chat vs. Inline Comments

The first draft is never the final. Learn how to refine designs precisely and efficiently

Core Concept

Claude Design gives you two ways to iterate. Chat is for broad structural changes — redesigning a section, shifting the aesthetic, adding new parts. Inline comments are for clicking directly on a specific element on the canvas and requesting a targeted change. Knowing which to use and when is the skill that separates fast users from slow ones.

Task 06

Practice 10 Chat-Level Changes

Take the landing page from Week 1 and use the chat to make the following changes one at a time. After each change, look at what updated on the canvas before sending the next message.

🟢 Chat Prompts to Run (in order)
  • "Make the color scheme darker and more minimal."
  • "Rearrange so the about section comes before the services."
  • "Change the layout from single column to two columns for the services section."
  • "Add a testimonials section between services and contact."
  • "Make the hero section take up the full viewport height."
  • "Show me 2 alternative color palettes for this page."
  • "Change the font style to something more editorial and refined."
  • "Add a sticky navigation bar at the top."
  • "Make the contact form more prominent — full-width with a background color."
  • "Remove the skills cards section entirely and replace it with a bullet list of services."
✓ Done When
All 10 changes applied. You can see the design evolve meaningfully with each message.
Task 07

Practice 10 Inline Comment Edits

Click directly on elements on the canvas and leave inline comments for these specific, component-level changes. Inline comments are for precision — use them when you know exactly what to fix and where it is.

🟢 Inline Comment Examples
  • Click a button → "Increase the padding on this button."
  • Click a heading → "Make this heading larger and bolder."
  • Click a form field → "Change this to a dropdown instead of a text input."
  • Click a card → "Add a subtle border and shadow to this card."
  • Click an icon → "Replace this with a more minimal icon."
  • Click the nav → "Make this navigation background transparent."
  • Click a section → "Add more padding above and below this section."
  • Click body text → "Change this text to a lighter gray — it's too dark."
  • Click a CTA → "Use the primary brand color for this button."
  • Click the footer → "Make this footer simpler — just copyright and links."
⚠ Known Issue
Inline comments sometimes disappear before Claude reads them. If this happens, paste the comment directly into the chat instead. This is the official Anthropic workaround.
Task 08

Build the Chat vs. Comments Cheat Sheet

Based on your experience in Tasks 06 and 07, create a personal one-page reference that lists: when you use chat, when you use inline comments, and 5 of your most effective prompt patterns for each. This becomes a go-to reference for all future sessions.

💡 Rule of Thumb
If the change requires context, explanation, or affects more than one element → use chat. If you can point at it and describe it in one sentence → use inline comments.
Task 09

The Full Iteration Challenge

Build a design from scratch, then do 20 iterations on it — alternating between chat and inline comments. Goal: transform a rough first draft into something you'd be genuinely proud to show someone. Track how many iterations it takes before you feel the output is "good enough." Most people are surprised by how much iterations improve quality.

✓ Done When
You have a before/after screenshot of the same design, significantly improved through iteration.
W3

Adding Context, References & Brand Inputs

Learn to feed Claude visual references, brand assets, and existing designs to dramatically improve output quality

Core Concept

Context is everything in Claude Design. You can attach screenshots of existing designs, competitor products, wireframes, slide decks, and even code repositories to a project. Claude reads all of it and uses it to produce more relevant, on-brand, production-ready output from the very first draft.

Task 10

The "Make It Look Like This" Challenge

Find 3 websites or apps whose design you admire. Take a screenshot of their hero section or main page. Upload each screenshot into a new Claude Design project and use this prompt structure:

🟢 Reference Prompt Template
"Here's a screenshot of a design style I want to reference. Create a [type of page] for [your brand/project] in a similar visual direction — same level of refinement, similar use of whitespace, similar typographic hierarchy. But make the content completely original."

Do this 3 times with 3 different references. Notice how dramatically the visual quality improves when Claude has something to work from.

Task 11

Build Your Personal Brand Context File

This is one of the most important tasks in Month 1. Create a document (plain text is fine) that contains your complete brand context. You will paste this at the start of every Claude Design project going forward. Include all of the following:

  • Brand name and one-sentence description
  • Primary color (hex code), secondary color, accent color
  • Primary font, secondary font (if applicable)
  • Tone of voice (e.g., "Professional but warm, never formal")
  • Target audience (age, profession, pain points)
  • What the brand should never look like
  • 3 brands whose aesthetic you admire and want to reference
  • Any specific components you always use (e.g., rounded cards, flat buttons)
💡 How to Use It
Start every new Claude Design session with: "Here is my brand context: [paste file]. Keep all of this in mind throughout our session."
Task 12

The 3-Brand Inspiration Challenge

Pick 3 brands you love. Screenshot their hero sections. Create a new fictional brand and have Claude generate a hero section inspired by each of the 3 references — separately, in 3 different projects. Compare the outputs. This shows you how much reference material shapes the output and trains you to give better visual direction.

✓ Done When
You have 3 dramatically different hero sections for the same fictional brand, each influenced by a different reference.
Task 13

Upload an Existing Slide Deck or Document

Find an existing presentation, report, or document you've created. Upload it into a Claude Design project and use this prompt:

🟢 Style Replication Prompt
"I've uploaded an existing document. Analyze its visual style — the fonts, colors, spacing, and layout approach. Now create a new [landing page / one-pager / presentation] that follows the same visual language but for a new topic: [describe topic]."

🟢 Version Save Prompt

Repeat this 3 times in the same project so you have 3 different saved versions. Practice navigating back to earlier versions in the conversation. This is your undo button.

Task 16

Ask Claude to Review Its Own Design

After generating any design, ask Claude to critique it. This is one of the most underused features — Claude can act as a design reviewer, not just a generator.

🟢 Design Review Prompts
  • "Review this design for accessibility issues. Check contrast ratios and text size."

W5

Presentations & Pitch Decks

Build professional slide decks in under 20 minutes — the highest-ROI skill in Claude Design

Task 18

Build a Full 10-Slide Pitch Deck

Create a complete pitch deck for a real or fictional business. Use this structured prompt that tells Claude exactly what slides to include:

🟢 Pitch Deck Master Prompt
"Design a 10-slide pitch deck for [business name]. The slides should be: 1) Cover slide with logo and tagline, 2) Problem — what pain point do we solve, 3) Solution — how we solve it, 4) How it works — product overview in 3 steps, 5) Market size — TAM/SAM/SOM, 6) Traction — key metrics and milestones, 7) Business model — how we make money, 8) Team — founders and key roles, 9) Roadmap — next 12 months, 10) Ask — funding amount and use of funds. Make it clean, investor-ready, and modern. Use a dark color scheme with bold typography."
✓ Done When
A complete 10-slide deck is generated. Then do at least 8 iterations to polish it.
Task 19

The Alternative Styles Challenge

After generating your pitch deck, ask for completely different visual styles to compare:

🟢 Style Variation Prompts
  • "Show me a minimal, white-background version of this deck — Apple-style."
  • "Now give me a bold, dark, high-contrast version with large typography."
  • "Try a warm, earthy palette — beige, tan, and deep green."

Compare all 3 side-by-side. Notice how the same content reads completely differently with different visual styles. Choose the one that best fits the brand personality.

Task 20

The Full Canva Handoff Workflow

Practice the complete flow from Claude Design to Canva to published presentation. This is the workflow you'll use with clients.

  • Finish your pitch deck in Claude Design
  • Click Export → Send to Canva
  • Open in Canva — verify all slides transferred correctly
  • Make final text edits and polish directly in Canva
  • Add real company logo and images in Canva
  • Export as PDF or share a Canva link
💡 Workflow Note
Claude Design is for the initial build and structural iteration. Canva is for final polish, real images, and publishing. They work as a two-step pipeline.
Task 21

Speed Goal: A Full Deck in Under 20 Minutes

Set a timer. Start from a blank project. Use your brand context file and your best prompt structure from Task 18. Build and iterate a complete presentation deck. Stop at 20 minutes. Evaluate what you produced. Repeat until you consistently hit the 20-minute target. This builds real-world speed.

Prototypes & Interactive Designs

Build interactive product prototypes that can be handed off directly to engineering

Why This Matters

Claude Design's interactive prototype capability is one of its most powerful features. Complex pages that took 20+ prompts in other tools only require 2 prompts in Claude Design. And when you're done, the Claude Code handoff packages everything your engineering team needs — no design spec documents required.

Task 22

Build a Mobile App Onboarding Prototype

Create an interactive mobile prototype with multiple screens and button states:

🟢 Prototype Prompt
"Design a mobile app onboarding flow with 4 screens. Screen 1: Welcome screen with app name, tagline, and 'Get Started' button. Screen 2: Account creation form with name, email, and password fields. Screen 3: Feature highlights — 3 cards showing the main value props with icons. Screen 4: Permission requests — location and notifications with Accept/Skip options. Make it interactive — the buttons should navigate between screens. Use a clean, modern mobile UI style."

After the prototype is generated, test clicking through the screens. Then iterate: change the visual style, add a loading screen, or adjust the copy.

Task 23

Build a Form with Conditional Logic

This tests Claude Design's ability to handle interactive UI logic:

🟢 Conditional Form Prompt
"Design a customer feedback form. The first question is: 'What is the nature of your feedback?' with options: Compliment, Issue, Feature Request, General. If the user selects 'Issue', show a follow-up field: 'Describe the issue and when it occurred.' If they select 'Feature Request', show: 'What would this feature help you accomplish?' The other options just show a general text field. Include a star rating for overall experience and a submit button."
Task 24

Practice the Claude Code Handoff

Take a completed prototype and practice the complete handoff to Claude Code. This is how design becomes production-ready code:

  • Complete any interactive design in Claude Design
  • Click Export → Handoff to Claude Code
  • Choose "Send to Claude Code Web" (no setup required)
  • Review the handoff package — note what Claude includes
  • If you have Claude Code installed, pass it with the prompt: "Build this exactly as designed"
💡 Pro Note
Claude packages design intent alongside the files — not just assets. This means the engineering handoff includes reasoning about spacing, interaction states, and component hierarchy.
Task 25

Get Real User Feedback on a Prototype

Share one of your interactive prototypes with someone who has never seen it. Use the organization sharing feature (view-only link). Watch them interact with it. Note: what do they click first? Where do they get confused? What do they say? Take their feedback and run at least 5 more iterations to address what you learn. This is real UX research.

W8

Design System Integration

Connect your brand's design system to every project so outputs are always on-brand, automatically

Core Concept

Claude Design can read your company's codebase and design files to understand your existing components, architecture, and styling patterns. When you link these, every design Claude generates automatically follows your brand system — the right colors, fonts, spacing, and components — without you needing to specify them each time.

Task 30

Document Your Own Design System

Before you can integrate a design system, you need to articulate it clearly. Create a "Design System Document" with these sections:

  • Color tokens: Primary, secondary, accent, error, success, background, text
  • Typography scale: H1, H2, H3, body, caption — with font name and size
  • Spacing rules: your base spacing unit and how you multiply it
  • Border radius: default radius for buttons, cards, inputs
  • Shadow levels: subtle, medium, strong
  • Component inventory: list every UI component your brand uses by name
  • What to never do: color combos, fonts, patterns that are off-brand
Task 31

Build 3 Outputs from the Same Design System

Paste your full design system document at the start of a session, then build 3 completely different outputs — all must stay within the system:

🟢 Design System Session Prompt
"Here is our complete design system: [paste system]. For every design you create in this session, strictly follow these guidelines. Do not deviate from the colors, fonts, or component styles described. Now: 1) Create a dashboard homepage, 2) Create a mobile settings screen, 3) Create a marketing landing page. All three should feel like they're from the same product family."
Task 32

The Design System Audit Prompt

After generating any design, use this prompt to have Claude check its own compliance:

🟢 Compliance Check Prompts
  • "Review this design against the design system I provided. List every element that breaks the guidelines."
  • "This button is using the wrong color — which component in our system should it use?"
  • "Are the spacing values on this page consistent with our 8px grid system?"
  • "What components from our system are missing that would make this feel more on-brand?"
Task 33

Link a GitHub Repository (Team/Enterprise)

If you're on a Team or Enterprise plan and have an existing codebase, practice linking it to a Claude Design project. Claude will read your actual components and produce designs that match your real production code — not just a style reference.

⚠ Known Limitation
Linking very large monorepos can cause lag or browser issues. Link specific subdirectories rather than the entire repository root.

Client-Ready Workflows

Build a Reusable Client Briefing Prompt Template

Create a master template you use at the start of every new client project. It should pull in everything Claude needs to deliver on-brand work from the first draft. Here's the structure:

🟢 Client Briefing Template Structure
"Client: [name] Industry: [industry] Target audience: [description] Brand colors: [hex codes] Typography: [font names] Tone of voice: [description] Competitor designs to NOT look like: [names] 3 brands whose aesthetic we admire: [names] Project: [what we're building] Key message: [one sentence] Most important section: [name it] Deliverable format: [PDF / PPTX / HTML / Canva] Deadline: [date]"

Save this as your master template. Fill it in before every new client project. Paste it as your first message in every new Claude Design session.

Task 35

Build a Complete Client Deliverable — Start to Finish

This document becomes part of every client deliverable — it positions you as a thoughtful designer, not just someone using a tool.

Task 43

Use Claude as a Design Critic — Go Deep

Take your best design from the past 3 months. Run all of these critique prompts on it. Apply every piece of feedback Claude gives. This is a masterclass in design thinking:

🟢 Advanced Critique Prompts
  • "What are the 5 biggest design problems with this? Be direct and specific."
  • "Review the visual hierarchy. What does the user's eye do when they land on this page?"
  • "Is this design accessible? Check contrast ratios, touch target sizes, and text legibility."

  • "If a new user lands on this for the first time, what are they confused by?"
  • "What would make this convert better? Give me 5 specific CRO (conversion rate optimization) suggestions."
